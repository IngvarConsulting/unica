# Platform 8.3.27 / Export Format 2.20 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Make `8.3.27 / 2.20` Unica's only writable platform-XML profile, block unsupported mutations before the first write, correct evidenced writer deviations, and provide explicit CF/CFE migration tools for older exports.

**Architecture:** Add a pure domain format profile and an application-level format guard parallel to the support guard. Native XML operations declare their format-guard policy in operation descriptors; read-only calls receive structured compatibility diagnostics while incompatible mutations are blocked. Explicit migration tools orchestrate the existing typed `v8-runner` adapter through disposable staging and publish only a verified `2.20` tree.

**Tech Stack:** Rust 2021, `roxmltree`, `serde`/`serde_json`, existing `ProcessRunner` and `RuntimeAdapter`, Python 3.12 CI contract tests, 1C:Enterprise 8.3.27 for ignored platform roundtrip tests.

## Global Constraints

- The only active platform profile is line `8.3.27` with configuration export format `2.20`.
- Missing root `version` means `1.0`; any source `< 2.20` is older and migration-eligible.
- A source `> 2.20` is unsupported; never offer or perform a downgrade.
- No `add`, `edit`, `compile`, `remove`, support mutation, or migration may write before full preflight succeeds.
- Read-only `info` and `validate` operations continue where safe and report format incompatibility separately.
- Migration is never automatic; only `unica.cf.migrate_format` or `unica.cfe.migrate_format` may start it.
- Do not replace the `version` attribute as a migration technique; load and re-dump through platform 8.3.27.
- Keep the public MCP boundary to one server named `unica` with `unica.*` tools.
- Do not expose direct packaged-script or raw Designer execution paths in prompt-visible skills.
- Migration must resolve an installed `8.3.27.*` platform, set `V8_PATH` only for its child runtime processes, and verify the reported platform version before loading source XML; an ambient 8.5/default platform is never accepted.
- XSD is additional evidence; successful platform 8.3.27 load/roundtrip is the final compatibility proof.
- Do not commit `docs-local`, downloaded XSD corpora, credentials, connection strings, disposable infobases, or staging trees.

pub const ACTIVE_FORMAT_PROFILE: FormatProfile = FormatProfile {
    platform_line: "8.3.27",
    export_format: ExportFormatVersion::new(&[2, 20]),
};

pub fn classify_root_version(raw: Option<&str>) -> Result<FormatCompatibility, FormatVersionError> {
    let actual = match raw {
        None => ExportFormatVersion::new(&[1, 0]),
        Some(value) => ExportFormatVersion::parse(value)?,

        Ordering::Greater => FormatCompatibility::Newer { actual },

fn resolve_version_root(target: &Path) -> Result<Option<PathBuf>, String> {
    for ancestor in target.ancestors() {
        let configuration = ancestor.join("Configuration.xml");
        if configuration.is_file() { return Ok(Some(configuration)); }
    }
    if target.is_file() && is_versioned_platform_root(target)? {

    }
    Ok(None)
}
