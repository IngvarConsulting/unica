# Investigation Archives

This directory contains tracked, non-normative research and handoff material.
It exists so unfinished investigations can survive branch switches and later be
continued without turning provisional conclusions into active requirements.

Authority remains, in order:

1. code, tests, and package metadata;
2. active `spec/` documents and accepted ADRs;
3. historical implementation plans;
4. these investigation archives.

An investigation is not proof that a feature ships, a design was accepted, or
a test currently passes. A conclusion becomes normative only after it is
promoted into its owning spec or ADR and backed by current code/tests.

Start with the project-specific index and read only the named documents. These
archives can be large and may intentionally preserve rejected alternatives,
stale paths, contradictory reviews, and incomplete prototypes.
