# Unica v0.7.0

> Source-only release: external marketplace staging found a Windows MCP startup
> stack overflow before catalog promotion. The immutable tag and assets were
> retained, not overwritten. `v0.7.1` corrected that runtime entrypoint but a
> second staging smoke found the same Windows stack constraint in the native
> bootstrap. `v0.7.3` supersedes both source-only releases.

Unica 0.7.0 changes the delivery model from platform-sized marketplace bundles
to a public thin Git marketplace. This note describes the release contract; the
release is considered published only after the source release, runtime bytes,
marketplace tag, stable catalog, and fresh-install smoke have all been verified.

## Install and prerequisites

Consumers need a compatible Codex CLI and standard Git (Git for Windows on
Windows). Node.js, Python, HTTP download utilities, JSON utilities, and archive
utilities are no longer installation prerequisites.

```sh
codex plugin marketplace add IngvarConsulting/unica-marketplace --ref main
codex plugin add unica@unica
```

Open a new Codex task after installation.

## Runtime artifacts

The source release publishes:

- `unica-runtime-darwin-arm64.tar.gz` and matching JSON metadata;
- `unica-runtime-linux-x64.tar.gz` and matching JSON metadata;
- `unica-runtime-win-x64.tar.gz` and matching JSON metadata;
- transition shims `install-unica.sh` and `install-unica.ps1`.

The marketplace package contains only plugin content and three small native
bootstraps. The current host runtime is downloaded on first MCP launch, checked
against archive and per-file SHA-256 values, and atomically cached under
`$CODEX_HOME/unica/runtimes/0.7.0/<target>`.

## Migration and rollback

Transition shims use Git to fetch the stable catalog and invoke the same native
migration engine on POSIX and Windows. Preflight runs before mutation. Successful
steps are journaled; on failure the engine executes inverse Codex commands and
atomically restores the exact saved config. Backups remain under
`$CODEX_HOME/unica/migration-backups/` and their path is printed.

## Update and uninstall

```sh
codex plugin marketplace upgrade unica
codex plugin remove unica@unica
codex plugin add unica@unica
```

Open a new Codex task after update. To uninstall:

```sh
codex plugin remove unica@unica
codex plugin marketplace remove unica
```

## Publication proof

Release acceptance requires the signed source tag, successful three-platform
workflow, re-downloaded runtime hashes, plugin-only marketplace staging merge,
signed immutable marketplace tag, catalog-only promotion merge, and isolated
Codex MCP `initialize`/`tools/list` proof on all supported platforms.
