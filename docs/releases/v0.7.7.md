# Unica v0.7.7

`v0.7.7` — технический релиз. Он исправляет удаление нового плагина во время
очистки старого cache, но не является поддерживаемым мостом миграции. Для
перехода со старых установок используйте `v0.7.8`.

## Что было исправлено

- Пересекающийся старый путь удаляется до установки нового `unica@unica`.
- Остальные старые пути удаляются только после проверки runtime, MCP и
  доступных в prompt навыков.
- Контракт #90 одновременно создаёт `plugins/cache/unica-local` и
  `plugins/cache/unica/unica/0.6.1` и проверяет, что новая установка сохраняется.
- Откат возвращает точное состояние, снятое перед миграцией.

## Почему релиз технический

Версионный тег marketplace создавался до продвижения стабильного каталога.
Поэтому тег `v0.7.7` содержал файлы плагина `0.7.7`, но каталог предыдущей
версии. Кроме того, bootstrap заменял выбранный ref на `main` при вызове Codex
CLI. Оба дефекта устранены в `v0.7.8`.

## Что использовать

Для перехода со старой установки используйте скрипты релиза `v0.7.8`. Уже
установленная через публичный marketplace версия `v0.7.5` или новее обновляется
обычными командами Codex CLI.
