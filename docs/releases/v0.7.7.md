# Unica v0.7.7

Unica `v0.7.7` is the corrected immutable migration bridge for historical
local, `unica-local`, duplicated, and canonical marketplace installations.
It supersedes `v0.7.6`, whose manual full-history barrier exposed an overlap
between a captured legacy cache path and the destination of the newly installed
canonical plugin.

## Migration fix

The transaction now separates captured legacy paths into two groups:

- a legacy path that contains the canonical plugin destination is removed
  before Codex installs the new package;
- every non-overlapping legacy path remains untouched until runtime, MCP tools,
  and prompt-visible skills have been verified.

This preserves the existing late-cleanup guarantee while preventing the final
cleanup from deleting the package installed by the same transaction. A focused
issue #90 contract materializes both `plugins/cache/unica-local` and
`plugins/cache/unica/unica/0.6.1`, then proves that only the new canonical
package remains.

## Supported transition

Legacy installations of any reported version must use these frozen assets:

- [`install-unica.sh`](https://github.com/IngvarConsulting/unica/releases/download/v0.7.7/install-unica.sh)
- [`install-unica.ps1`](https://github.com/IngvarConsulting/unica/releases/download/v0.7.7/install-unica.ps1)

Canonical `v0.7.5`, `v0.7.6`, `v0.7.7`, and canonical technical `0.7.x`
installations can use the ordinary marketplace update path to `v0.8.0`.
Starting with `v0.8.0`, direct legacy migration is no longer part of the current
Unica package; these immutable `v0.7.7` assets remain the supported bridge.

## Verification ownership

The source repository owns the focused transaction contracts. The marketplace
repository owns the dispatch-only full-history regression across macOS, Linux,
Windows, representative historical Codex CLI versions, idempotence, prompt/MCP
visibility, settings preservation, and injected rollback. Issue #90 closes only
after that workflow succeeds against the published and promoted `v0.7.7` bytes.
