# Unica v0.7.3

Unica 0.7.3 completes the public marketplace migration contract. It keeps the
thin, Node-free runtime delivery introduced in 0.7.2 and fixes cleanup of
legacy local installations whose inactive state is no longer returned by Codex
JSON discovery.

## Install and prerequisites

Consumers need a compatible Codex CLI and standard Git (Git for Windows on
Windows). Node.js, Python, HTTP download utilities, JSON utilities, and archive
utilities are not installation prerequisites.

```sh
codex plugin marketplace add IngvarConsulting/unica-marketplace --ref main
codex plugin add unica@unica
```

Open a new Codex task after installation.

## Migration cleanup and rollback

Preflight now combines Codex JSON discovery with inspection of the two exact
documented legacy cache locations and the old plugin config table. This catches
orphaned state even when current Codex output normalizes or omits it.

Before any mutation, the transaction copies the exact config and identified
legacy directories into `$CODEX_HOME/unica/migration-backups/`. It removes the
old plugin registration through Codex CLI, proves the canonical plugin, deletes
only the identified legacy paths after checksum-pinned runtime installation and
MCP initialize/tools-list verification, and proves the final state again. Any
failure restores the saved config and directories. A completed migration is
idempotent.

## Runtime and update

The marketplace remains thin and contains three native bootstrap binaries. The
first MCP start downloads only the current host runtime, verifies its archive
and per-file SHA-256 values, then atomically caches it under
`$CODEX_HOME/unica/runtimes/0.7.3/<target>`.

```sh
codex plugin marketplace upgrade unica
codex plugin remove unica@unica
codex plugin add unica@unica
```

Open a new Codex task after update.
