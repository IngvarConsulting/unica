# Unica v0.7.6

Unica `v0.7.6` was intended to be the final immutable migration bridge, but its
manual full-history barrier found that cleanup could delete a newly installed
canonical package when legacy and canonical cache roots overlapped. The signed
release remains immutable, but it is superseded by `v0.7.7` and must not be used
as the legacy migration bridge.

## Migration fixes

The native preflight now keeps the marketplace's reported identity separate
from its managed filesystem path. This covers both historical issue #90 states:

- a legacy marketplace registered as `unica-local`;
- a legacy marketplace registered as `unica` while its managed path remains
  `marketplaces/unica-local`.

The transaction removes only owned legacy registrations and paths, installs
exactly one canonical `unica@unica`, and preserves the complete user-owned
canonical settings subtree across Codex remove/add mutations. A failed step
restores the exact preflight `config.toml`, registrations, cache paths, file
permissions, and saved content.

## Supported transition

Legacy installations of any reported version must use the corrected frozen
installers published by `v0.7.7`:

- [`install-unica.sh`](https://github.com/IngvarConsulting/unica/releases/download/v0.7.7/install-unica.sh)
- [`install-unica.ps1`](https://github.com/IngvarConsulting/unica/releases/download/v0.7.7/install-unica.ps1)

Canonical `v0.7.5`, canonical `v0.7.6`, and canonical technical `0.7.x`
installations can use the ordinary marketplace update path to `v0.8.0`.
Starting with `v0.8.0`, direct legacy migration is no longer part of the current
Unica package; the immutable `v0.7.7` assets remain the supported bridge.

## Verification ownership

The source release keeps focused contract and preflight tests for both issue
#90 path identities. The marketplace repository owns the manual full-history
regression across macOS, Linux, Windows, the representative historical Codex
CLI, settings preservation, idempotence, MCP/prompt visibility, and injected
rollback. Issue #90 closes only after that manual workflow succeeds against the
published and promoted `v0.7.7` bytes.
