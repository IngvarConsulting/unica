# Unica v0.7.6

`v0.7.6` — технический релиз. Он не является поддерживаемым мостом для перехода
со старых установок; используйте `v0.7.8`.

## Что было исправлено

- Предварительная проверка различает имя marketplace и его управляемый путь.
- Учтены оба состояния из #90: `unica-local` и регистрация `unica`, оставшаяся
  в старом каталоге `marketplaces/unica-local`.
- Пользовательские настройки `unica@unica` сохраняются при удалении и повторной
  установке через Codex CLI.
- При ошибке восстанавливаются исходный `config.toml`, регистрации, права и
  сохранённые каталоги.

## Почему релиз технический

Ручная полная проверка обнаружила пересечение старого cache-пути с каталогом
нового плагина: финальная очистка могла удалить только что установленную Unica.
Исправление этого сценария вошло в следующую техническую версию, но окончательный
стабильный барьер миграции выпущен только в `v0.7.8`.

## Что использовать

Для перехода со старой установки используйте скрипты релиза `v0.7.8`. Уже
установленная через публичный marketplace версия `v0.7.5` или новее обновляется
обычными командами Codex CLI.
