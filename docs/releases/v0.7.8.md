# Unica v0.7.8

`v0.7.8` — стабильный мост для перехода со старых установок Unica на публичный
marketplace. Версии `v0.7.6` и `v0.7.7` признаны техническими и не должны
использоваться для миграции.

## Главное

- Исправлена причина рассинхронизации тегов: marketplace-тег создаётся только
  для commit, в котором уже совпадают файлы плагина и стабильный каталог.
- Скрипт передаёт замороженный ref `v0.7.8` в native bootstrap.
- Bootstrap регистрирует marketplace с `main`, временно переключает управляемый
  checkout на `v0.7.8` и устанавливает через Codex CLI именно проверяемый мост.
- После проверки моста bootstrap выполняет обычный marketplace upgrade: checkout
  возвращается на `main`, а плагин обновляется до текущей стабильной версии.
- Сценарии #90 сохраняют пользовательские настройки и полностью откатываются
  при любой ошибке.

## Переход со старой версии

Для версий `0.3.0`–`0.7.4` используйте опубликованный скрипт `v0.7.8`:

```sh
curl -fLO https://github.com/IngvarConsulting/unica/releases/download/v0.7.8/install-unica.sh
sh install-unica.sh --ref v0.7.8
```

В Windows PowerShell:

```powershell
Invoke-WebRequest https://github.com/IngvarConsulting/unica/releases/download/v0.7.8/install-unica.ps1 -OutFile install-unica.ps1
.\install-unica.ps1 -Ref v0.7.8
```

Версия `0.7.5` или новее обновляется обычными командами Codex CLI.

## Проверка

В исходном репозитории выполняются контрактные и preflight-тесты. Полный
регресс миграции запускается вручную в репозитории marketplace: старый Codex
CLI, обе формы путей из #90, сохранение настроек, повторный запуск, MCP, навыки
в prompt и принудительная ошибка с полным откатом.
