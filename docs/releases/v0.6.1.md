## Unica v0.6.1

Этот релиз публикуется поверх `v0.5.1`. Отдельный GitHub Release `v0.6.0` не выпускался, поэтому все изменения ниже считаются относительно `v0.5.1`.

### Главное

- Продолжена миграция на MCP-first runtime: prompt-visible skills остаются за публичными `unica.*` tools, а legacy script fallback для форм и СКД удален из runtime-пути.
- Существенно расширены native-операции 1С в Rust runtime: формы, СКД, CFE, metadata, роли, шаблоны и support-state теперь покрывают больше практических сценариев без возврата к packaged scripts.
- Добавлен новый `support-edit` skill и native-инструмент для явного изменения состояния поддержки объектов конфигурации.
- Добавлена большая BSP parity-база: реальные fixtures и проверки для `cfe-borrow`, `cfe-diff`, `form-compile`, `form-compile-from-object`, `meta-edit`, `skd-compile` и связанных операций.
- Усилены архитектурные guardrails: единая публичная граница `unica`, запрет на возвращение прямых script-backed путей там, где уже есть native `unica.*` tool, и явные исключения для utility skills.

### Исправлено

- Windows MCP launcher после установки запускается из корректного plugin cwd и не теряет bundled binaries (#30).
- Native XML writer больше не генерирует UUID с некорректным нулевым префиксом; роли получают UUID v4 и сохраняют UUID при повторной генерации (#32, #39).
- `meta.edit` получил documented writer-операции для register records, attributes и tabular-section attributes; исправлены BOM, inline-колонки и сохранение `ChildObjects` (#37, #38).
- `form.edit` и `form.validate` закрывают parity gaps для command bar, контейнеров и duplicate bindings (#36, #44).
- `form.add` уважает `SetDefault=false`, а `form.remove` и round-trip форм больше не портят пустые теги родительского XML (#56, #57, #66).
- `cf.edit` сохраняет XML declaration, line endings и форматирование соседних `ChildObjects` при `add-childObject`/`remove-childObject` (#50, #55).
- `interface.edit` корректно обрабатывает hide/show для command interface, созданного через `CreateIfMissing` (#58).
- Windows path/source guardrails отклоняют выход `OutputDir` за пределы workspace и нормализуют evidence с Windows path separators (#61, #62).
- `runtime.execute` согласован с текущим контрактом `v8-runner`, получил более полезную диагностику процессов и больше не использует legacy `V8TR_CONFIG` (#47).

### Инструменты и инфраструктура

- Обновлены bundled tool pins: `bsl-analyzer v0.2.55`, `rlm-tools-bsl/rlm-bsl-index v1.26.0`; `v8-runner` остается на проверенном `v0.5.1`.
- Release workflow и package tests дополнены проверками installer/runtime contract, packaged `unica` binary, source guardrails и BSP release assessment.
- `package-unica-plugin.py`, `build-unica-tools.py` и installer tests стали строже к target-specific bundle contract и Windows/POSIX различиям.
- Web-test helpers получили доработки DOM/table/form automation для более надежной работы с 1С web-client сценариями.
