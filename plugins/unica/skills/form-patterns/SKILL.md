# /form-patterns — паттерны компоновки форм

Справочник типовых паттернов дизайна управляемых форм 1С. Используй **перед** проектированием формы через `unica.form.compile`, когда требования пользователя не детализируют расположение элементов.

**Как использовать:** выбери подходящий архетип, применяй конвенции именования, сверяйся с UX-правилами для элементов, при необходимости используй продвинутые паттерны.

---

## Архетипы форм

### Форма документа

```
Шапка (horizontal, 2 колонки)
├─ Левая (vertical): НомерДата (H: Номер + Дата "от"), Контрагент, Договор
├─ Правая (vertical): Организация, Подразделение, ЦеныИВалюта (надпись-ссылка)
Страницы (pages)
├─ Товары: таблица Объект.Товары
├─ Услуги: таблица Объект.Услуги (опционально)
└─ Дополнительно: прочие реквизиты
Подвал (vertical)
├─ Итоги (horizontal): Всего, НДС, Скидка
└─ КомментарийОтветственный (horizontal): Комментарий + Ответственный
```

**События:** OnCreateAtServer, OnReadAtServer, OnOpen, BeforeWriteAtServer, AfterWriteAtServer, AfterWrite, NotificationProcessing
**Свойства:** autoTitle=false

### Форма обработки (DataProcessor)

```
Параметры (vertical)
├─ Группа полей ввода (Организация, Период, режимы работы)
├─ Информационные надписи (label, hyperlink)
Рабочая область
├─ Таблица данных или Pages с вкладками
Кнопки действий
├─ Выполнить / Применить (defaultButton)
├─ Закрыть (stdCommand: Close)
```

**События:** OnCreateAtServer, OnOpen, NotificationProcessing
**Свойства:** windowOpeningMode=LockOwnerWindow (если диалог), autoTitle=false

### Форма списка

```
Отборы (group: alwaysHorizontal)
├─ ГруппаОтбор[Поле] (H): Флажок + Поле ввода (для каждого фильтра)
Список (table, DynamicList)
├─ Колонки: labelField (не input — данные только для чтения)
```

**События:** OnCreateAtServer, OnOpen, NotificationProcessing, OnLoadDataFromSettingsAtServer
**Свойства:** autoSaveDataInSettings=Use
**Фильтры:** пара реквизитов на каждый — `Отбор[Поле]` (значение) + `Отбор[Поле]Использование` (boolean)

### Форма элемента справочника

**Простая:**
```
ГруппаРеквизитов (horizontal)
├─ Наименование -> Объект.Description
└─ Код -> Объект.Code (если нужен)
```

**Сложная:**
```
Главное (vertical)
├─ Наименование -> Объект.Description
├─ Параметры (horizontal, 2 колонки)
│  ├─ Левая: основные реквизиты
│  └─ Правая: дополнительные реквизиты
└─ КонтактныеДанные / Дополнительно (vertical)
```

**События:** OnCreateAtServer, OnReadAtServer, BeforeWriteAtServer, NotificationProcessing

### Мастер (Wizard)

```
Страницы (pages, OnCurrentPageChange)
├─ Шаг1: описание + параметры
├─ Шаг2: основная работа
└─ Шаг3: результат
Кнопки (horizontal)
├─ Назад (command), Далее (command, defaultButton), Выполнить (command)
└─ Закрыть (stdCommand: Close)
```

**Свойства:** windowOpeningMode=LockOwnerWindow, commandBarLocation=None

## Конвенции именования

### Группы

| Назначение | Имя | Тип |
|-----------|-----|-----|
| Шапка | `ГруппаШапка` | horizontal |
| Левая колонка | `ГруппаШапкаЛевая` | vertical |
| Правая колонка | `ГруппаШапкаПравая` | vertical |
| Номер+Дата | `ГруппаНомерДата` | horizontal |
| Подвал | `ГруппаПодвал` | vertical |
| Итоги | `ГруппаИтоги` | horizontal |
| Кнопки | `ГруппаКнопок` | horizontal |
| Страницы | `ГруппаСтраницы` / `Страницы` | pages |
| Предупреждение | `ГруппаПредупреждение` | horizontal, visible:false |
| Доп. секция | `ГруппаДополнительно` / `ГруппаПрочее` | vertical, collapse |

### Элементы

| Назначение | Имя |
|-----------|-----|
| Поле в таблице | `[Таблица][Поле]` |
| Итог | `Итоги[Поле]` |
| Надпись-ссылка | `[Поле]Надпись` |
| Фильтр | `Отбор[Поле]` |
| Флажок фильтра | `Отбор[Поле]Использование` |
| Кнопка команды | `[Команда]Кнопка` |
| Баннер-картинка | `[Баннер]Картинка` |
| Баннер-надпись | `[Баннер]Надпись` |
| Подменю | `Подменю[Действие]` |

### Обработчики событий

Имя = имя элемента + суффикс на русском:

| Событие | Суффикс | Пример |
|---------|---------|--------|
| OnChange | ПриИзменении | `ОрганизацияПриИзменении` |
| StartChoice | НачалоВыбора | `КонтрагентНачалоВыбора` |
| Click | Нажатие | `ЦеныИВалютаНажатие` |
| OnEditEnd | ПриОкончанииРедактирования | `ТоварыПриОкончанииРедактирования` |
| OnStartEdit | ПриНачалеРедактирования | `ТоварыПриНачалеРедактирования` |

Обработчики формы: `ПриСозданииНаСервере`, `ПриОткрытии`, `ПередЗакрытием`, `ОбработкаОповещения`.

## Принципы компоновки

1. **Порядок чтения.** Сверху вниз, слева направо. Самое важное — вверху.
2. **Двухколоночная шапка.** Основные реквизиты слева (контрагент, склад), организационные справа (организация, подразделение).
3. **Кнопки действий внизу.** Главная кнопка — `defaultButton: true`. Закрыть — всегда последняя.
4. **Таблицы — основная область.** Табличные части занимают большую часть формы, обычно на Pages.
5. **Итоги рядом с таблицей.** В подвале, горизонтальная группа, все поля readOnly.
6. **Фильтры — отдельная зона.** Над списком, alwaysHorizontal, пара «флажок + поле» на каждый фильтр.
7. **Скрытые элементы для состояний.** Баннеры, предупреждения — `visible: false`, показываются программно.
8. **Надписи-ссылки для диалогов.** `labelField` с `hyperlink: true` и событием Click.

## UX-правила для элементов форм

Правила проектирования отдельных элементов интерфейса. Применяй при создании или ревью форм через `unica.form.compile`.

### Декорация-надпись
- Без точки в конце одного предложения
- Не использовать для заголовка простых полей (теряется автофокус)
- Не использовать без полезного действия
- Не использовать для выравнивания — используйте группы

### Декорация-картинка
- Только из библиотеки картинок, не из файла
- В составе форматированной строки или отдельным элементом

### Поле ввода
- Ширина — под самое длинное значение из списка
- Кнопка очистки (x) если пустое значение допустимо (кроме отборов — там флажок)
- Подсказка ввода для пустого значения
- Кнопка выбора для строковых полей, если есть из чего выбрать (путь к файлу, наименования)
- Многострочное поле для строк > 100 символов (заголовок сверху)
- Сохранять выбранные пользователем значения повторяющихся полей
- Не показывать заголовок у очевидных полей (Организация, БанковскийСчет) — добавьте подсказку ввода
- Не использовать платформенный выбор для составных типов — добавьте отдельный выбор
- Не делать поле выбора для одного значения — заполните реквизит самостоятельно, а поле скройте функциональной опцией

### Флажок
- Заголовок справа (не по умолчанию платформы)
- Для простых опций с понятными обоими состояниями
- Не использовать для сложных опций, в которых не очевидны оба состояния — используйте переключатель
- Зона действия — небольшая (подсветка соседнего поля), для сильного влияния используйте выключатель
- Не использовать отрицательные формулировки в заголовке («Не Не проводить»)

### Выключатель (Тогл, Свитчер)
- Действие происходит сразу без подтверждения
- Ассоциируется с большим влиянием на форму, чем флажок
- Не больше 2-3 на странице

### Переключатель
- Для 2-5 значений; больше — поле ввода со списком
- Значение по умолчанию — первым в списке
- Подсказки к сложным значениям (создайте два элемента с путём к одному реквизиту)
- Повторяющийся текст выносить в заголовок группы
- Действует на то, что справа и под ним

### Тумблер
- Для 3-5 значений
- Если при выборе перестраивается форма (выглядит как кнопки)
- Компактнее переключателя, для коротких значений
- Не использовать в значениях повторяющиеся слова — вынесите их в заголовок
- Не использовать для 2 значений — неясно, какой выбран

### Гиперссылка
- Для открытия подчинённых и служебных форм, не для действий на форме
- Осмысленное название, понятно куда ведёт
- Не писать «здесь», «тут», «подробнее»
- Не ставить две ссылки подряд — трудно заметить переход
- Не менять цвет (кроме ссылок с предупреждением)

### Кнопка
- Для выполнения действия на форме, не для перехода к другой форме
- Понятный заголовок (не пытайтесь сделать коротким)
- Пометка для команд строки списка («Использовать как основной»)
- Без фигуры — для нерекомендуемых команд или когда кнопок слишком много
- Кнопка по умолчанию: `defaultButton: true`, жирный шрифт, золотой фон RGB(255,255,0)

### Табличная часть
- Заголовок в 1 строку (сложный заголовок — подсказки в строках таблицы)
- Убрать заголовок если одна колонка — описать декорацией
- Пустая последняя колонка: `readOnly: true`, `horizontalStretch: true`, заголовок отключён
- Прижимать заголовок колонки к значению (число — вправо, строка/ссылка — влево)

### Диалоговое окно
- Чёткий вопрос с однозначным ответом, подробное описание
- Текст кнопки повторяет запрашиваемое действие («Установить режим тишины», не «Да»)
- Обрабатывать закрытие по крестику — установить команду как `КодВозвратаДиалога.Отмена`

### Подсказка (Hint)
- Через кнопку (buttonHint) — для разовых подсказок новому элементу
- Сверху формы — для редко используемых форм (всегда перед глазами)
- Снизу элемента — для сложных реквизитов (объяснить значение)
- Разные средства оформления: желтый фон, картинки — для акцентирования
- Подсказок много не бывает, но они не должны быть навязчивыми

### Условное оформление

Визуальное выделение данных на основе заданных условий (conditionalAppearance):

- Применяйте одни и те же стили для аналогичных условий (например, новые элементы списка — жирным)
- Не используйте много разных стилей и цветов
- Не используйте условное оформление для управления видимостью строк

## Продвинутые паттерны (ERP)

### Сворачиваемые группы

Для необязательных секций (подписи, дополнительно, прочее):

```json
{ "group": "vertical", "name": "ГруппаПодписи", "title": "Подписи",
  "behavior": "Collapsible", "collapsed": true, "children": [...] }
```

### Баннер-предупреждение

Группа «картинка + надпись», скрыта по умолчанию, показывается программно:

```json
{ "group": "horizontal", "name": "ГруппаПредупреждение", "showTitle": false,
  "visible": false, "children": [
    { "picture": "ПредупреждениеКартинка" },
    { "label": "ПредупреждениеНадпись", "title": "Текст", "maxWidth": 76, "autoMaxWidth": false }
]}
```

### Popup-меню в командной панели

Группировка связанных команд (печать, отправка) в одну кнопку с иконкой:

```json
{ "cmdBar": "КоманднаяПанель", "children": [
    { "popup": "ПодменюПечать", "title": "Печать",
      "picture": "StdPicture.Print", "representation": "Picture", "children": [
        { "button": "ПечатьНакладная", "command": "Печать" },
        { "button": "ПечатьСчёт", "command": "ПечатьСчёт" }
    ]}
]}
```

### Форма без стандартной командной панели

Для модальных диалогов и мастеров:

```json
{ "properties": { "commandBarLocation": "None", "windowOpeningMode": "LockWholeInterface" } }
```

### Надпись-гиперссылка

Вместо кнопки для открытия подформ (ЦеныИВалюта, УчётнаяПолитика):

```json
{ "labelField": "ЦеныИВалютаНадпись", "path": "ЦеныИВалюта", "hyperlink": true, "on": ["Click"] }
```

## UX-правила для элементов форм

> Источник: [1C Design Guide](https://github.com/Oxotka/1CDesignGuide) (MIT).<br>
> Адаптация правил дизайна элементов форм — поля, флажки, кнопки, гиперссылки, таблицы, диалоги, подсказки и условное оформление.

### Поле ввода

1. Установите ширину поля такой, чтобы вмещалось самое длинное значение из списка
2. Добавьте кнопку очистки, если в поле ввода предусмотрено удаление или пустое значение (кроме отборов на формах — там используется флажок)
3. Опишите подсказку ввода для пустого значения на форме, если предусмотрено пустое значение
4. Добавьте кнопку выбора для строковых полей ввода, если есть из чего выбрать (путь к файлу, наименование)
5. Используйте многострочное поле ввода для строковых значений длиннее 100 символов. Заголовок многострочного поля установите сверху
6. Сохраняйте значения, выбранные пользователем для реквизитов формы, если они повторяются (период, организация)
7. Не показывайте заголовок у поля Организация или Банковский счет для сокращения места (например, в шапке отчета). Пользователь знает свою организацию
8. Добавляйте подсказку ввода, если не показываете заголовок. Для Организаций в отчете напишите «По всем организациям»
9. Не используйте платформенный выбор типа для составных типов. Добавьте отдельный выбор типа (Контрагент или Физическое лицо)
10. Не делайте поле выбора для одного значения — заполните реквизит самостоятельно, а поле скройте функциональной опцией

### Флажок

1. Установите заголовок флажка справа. Отойдите от умолчания платформы
2. Зона действия флажка небольшая или отсутствует. Например, при включении можно включить доступность соседнего реквизита или гиперссылки. Если требуется влиять на форму сильнее, используйте выключатель или переключатель
3. Используйте флажок для простой опции, в которой понятны оба состояния. Если из названия сложно понять действие, добавьте подсказку снизу
4. Не используйте флажок для сложных опций, в которых не очевидны оба состояния. Используйте переключатель
5. Не используйте в заголовке флажка отрицательные формулировки — пользователю сложнее понять выключенный флажок. Вместо «Не проводить документ при записи» напишите «Проводить документ при записи» и снимите флажок по умолчанию

### Выключатель (тогл, свитчер)

Выключатель переключает состояние формы, реквизита или настройки. Это вид флажка — к нему применяются правила флажка.

1. Используйте выключатель, если при переключении состояния действие происходит сразу и не нужны дополнительные подтверждения
2. Выключатель ассоциируется с большим влиянием, чем флажок. При изменении состояния действие на форму может быть более обширным
3. Не используйте на одной странице больше 2–3 выключателей

### Переключатель

1. Используйте переключатель, когда значений немного — от 2 до 5. Если больше, используйте поле ввода со списком значений
2. Установите значение по умолчанию первым в списке
3. Добавьте подсказки к значениям переключателя для сложных опций
4. Выносите повторяющиеся тексты значений переключателя в заголовок
5. Переключатель действует на то, что находится справа и под ним

### Тумблер

1. Используйте тумблер для 3–5 значений
2. Используйте тумблер, если от выбранных значений перестраивается форма под ним или выполняется действие — тумблер выглядит как кнопки
3. Тумблер компактнее переключателя и подходит для коротких значений
4. Не используйте в значениях тумблера повторяющиеся слова — вынесите их в заголовок
5. Не используйте тумблер для 2 значений — неясно, какой пункт выбран

### Гиперссылки

1. Используйте для открытия подчинённых и служебных форм (например, настройка НДС)
2. Давайте осмысленное и достаточно длинное название гиперссылке, чтобы в неё можно было удобно нажать
3. Из гиперссылки должно быть понятно, куда она ведёт
4. Не выполняйте действия на форме с помощью гиперссылки — используйте кнопки
5. Не пишите в гиперссылке слова «здесь», «тут» или «подробнее»
6. Не ставьте две ссылки подряд — трудно заметить переход между двумя ссылками
7. Не меняйте цвет гиперссылки, за исключением ссылок с предупреждением

### Кнопка

1. Используйте кнопку для выполнения действия по запросу пользователя
2. Используйте кнопку с пометкой для команды, связанной со строкой списка (табличная часть или динамический список), например «Использовать как основной»
3. Сделайте понятный заголовок для кнопки, не пытайтесь сделать его коротким. Если по кнопке открывается новая форма (не рекомендуется), название должно сообщать об этом: «Показать настройки»
4. Используйте кнопку без фигуры для команд, которые не рекомендуется выполнять, или когда кнопок слишком много. Применяйте с осторожностью
5. Не используйте кнопку для перехода к другой форме — используйте гиперссылку

### Табличная часть

1. Делайте заголовок табличной части в 1 строку. При необходимости вывести сложный заголовок используйте подсказки в самой таблице, в строке
2. Уберите заголовок табличной части, если в ней всего одна колонка. Опишите декорацией содержание, если это требуется
3. Добавьте в табличной части пустую последнюю колонку, чтобы управлять шириной. Отключите заголовок у этой колонки, включите «Только просмотр» и установите «Растягивать по горизонтали»
4. Прижимайте заголовок колонки табличной части к значению: для числа — вправо, для строки или ссылки — слева (по умолчанию)

### Диалоговое окно

Диалоговое окно — блокирующее сообщение, требующее обязательного внимания пользователя. Используйте с осторожностью.

1. Уделите особое внимание тексту в диалоговом окне. Опишите подробно, что требуется от пользователя, добавьте чёткий вопрос с однозначным ответом
2. Опишите утвердительное действие, чтобы пользователь однозначно понял, что будет выполнено. Установите текст этой кнопки так, чтобы он повторял запрашиваемое действие
3. Обрабатывайте закрытие диалогового окна по крестику справа вверху. Установите одну из команд диалогового окна как КодВозвратаДиалога.Отмена

### Подсказка и расширенная подсказка

1. Для разовых подсказок используйте отображение через кнопку. Если пользователь видит новый элемент и сомневается, что с ним делать, ему достаточно один раз посмотреть подсказку по кнопке
2. Используйте подсказку сверху о всей форме, если она используется нечасто. У пользователя такая подсказка будет всегда перед глазами
3. Сделайте подсказку снизу для сложных элементов, чтобы объяснить пользователю, что этот реквизит означает
4. Используйте различные средства оформления для акцентирования внимания на конкретный реквизит (жёлтый фон, картинки)
5. Подсказок много не бывает, но они не должны быть навязчивыми

### Условное оформление

1. Применяйте одни и те же стили для аналогичных условий (например, в динамическом списке новые элементы выделяются жирным)
2. Не используйте много различных стилей и цветов
3. Не используйте условное оформление для управления видимостью строк

### Правила для Unica Form DSL

При генерации формы через unica.form.compile учитывайте:

1. **Цвета** указывайте в формате "web:Yellow", "#RRGGBB" или "auto". Не используйте RGB(r,g,b) — это невалидный формат для DSL.
2. **Подсказки** задавайте через tooltip (строка) и tooltipRepresentation (например, "Button", "Top", "Bottom"). Не используйте buttonHint — это поле не поддерживается DSL.
3. **Кнопка по умолчанию** — ровно одна на форму. Установите defaultButton: true на одной команде. Если требуется визуально выделить несколько равнозначных кнопок, используйте оформление: font.bold: true и backColor: "web:Yellow" без defaultButton.
4. **Заголовки флажков** — без отрицательных формулировок. Вместо «Не проводить» → «Проводить» (снятый флажок = выключено).
5. **Обработка закрытия диалога** — устанавливайте returnCode: "Cancel" на одной из кнопок, чтобы обработать закрытие по крестику.
